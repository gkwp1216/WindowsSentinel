===============================================================================
                    Windows Sentinel - LogCheck 기능 설명서
===============================================================================

프로젝트 개요:
Windows Sentinel은 Windows 시스템의 보안 모니터링, 네트워크 분석 및 악성 프로그램 탐지를 위한 
종합적인 보안 솔루션 WPF 애플리케이션입니다. 실시간 위협 탐지와 예방을 통해 
개인과 기업의 디지털 자산을 안전하게 보호합니다.

개발 환경:
- 플랫폼: .NET 8.0 WPF
- 언어: C#
- UI 프레임워크: Material Design in XAML
- 차트 라이브러리: LiveCharts2
- 패킷 캡처: SharpPcap

   │     • 🔍 전체 시스템 검사 (설치된 프로그램 중심)                        │

   │       - 각 프로그램의 핵심 실행 파일(.exe) 악성코드 탐지                │

   │        • Windows 레지스트리에서 설치 프로그램 정보 자동 수집            │
   │        • 32비트/64비트 프로그램 모두 포함하여 완전한 목록 생성           │
   │        • 프로그램명, 설치경로, 제작사 정보 함께 수집                    │
   │                                                                         │
   │     2️⃣ 핵심 실행파일 식별                                              │
   │        • 각 프로그램의 설치 폴더에서 대표 실행파일(.exe) 자동 탐지      │

   │        • 파일 크기가 가장 큰 실행파일을 대표파일로 지정                 │
   │                                                                         │
   │     3️⃣ 디지털 지문 생성                                                │
   │        • SHA-256 암호화 해시 기술로 파일의 고유 지문 생성               │
   │        • 파일 내용이 1비트라도 변경되면 완전히 다른 지문 생성            │

   │                                                                         │
   │     4️⃣ 국제 악성코드 DB 조회                                           │
   │        • 생성된 지문을 MalwareBazaar 국제 데이터베이스에 조회           │
   │        • 전 세계 보안 전문가들이 수집한 악성코드 정보와 대조             │

   • 초기 설정 자동 진행 (감사 정책, Npcap 확인)

2. 네트워크 모니터링
   • 네트워크 탭 선택
   • 원하는 인터페이스 선택 후 캡처 시작
   • 실시간 패킷 분석 및 필터링

   • 사용 기록 탭 선택
   • 날짜 범위 및 필터 조건 설정
   • 조회 버튼 클릭하여 데이터 분석

4. 차트 분석
   • 차트 기간 및 유형 선택
   • 실시간 업데이트되는 시각화 데이터 확인

   • 백신 탭 선택
   • "전체 검사" 버튼으로 시스템 전체 검사
   • "파일 검사" 버튼으로 개별 파일 검사
   • 검사 결과 색상으로 위험도 즉시 확인

   • 복구 탭 선택
   • 현재 보안 상태 자동 진단 확인
   • "정밀 보안 진단 마법사" 버튼으로 전체 복구 실행
   • 각 서비스별 진행률 및 결과 실시간 확인
   • 복구 완료 후 시스템 재부팅 권장

===============================================================================
                              문제 해결
===============================================================================

일반적인 문제:
1. "관리자 권한 필요" 오류
   → 마우스 우클릭 → "관리자 권한으로 실행" 선택

2. 네트워크 캡처 실패
   → Npcap 설치 확인
   → 네트워크 어댑터 드라이버 업데이트

3. 차트가 표시되지 않음
   → 데이터 필터 조건 확인
   → 테스트 데이터 생성 버튼 활용

4. 프로그램 목록 오류
   → Windows 레지스트리 권한 확인
   → 시스템 재시작 후 재시도

5. 악성 프로그램 검사 실패
   → 인터넷 연결 상태 확인 (온라인 DB 조회 필요)
   → 방화벽에서 LogCheck 허용 설정
   → 바이러스 백신에서 LogCheck 예외 처리

6. "파일을 열 수 없습니다" 오류
   → 검사할 파일이 다른 프로그램에서 사용 중인지 확인
   → 파일 권한 및 관리자 권한으로 실행 확인
   → 파일 경로에 특수문자나 한글이 있는지 확인

7. 악성코드 검사 속도가 너무 느림
   → 검사할 프로그램 수가 많으면 시간이 오래 걸림 (정상)
   → 인터넷 연결 속도 확인 (실시간 온라인 조회 방식)
   → 동일한 프로그램 재검사 시에는 캐시로 빠른 결과 제공

8. "Rate Limit" 또는 "API 제한" 오류
   → MalwareBazaar 서비스의 일시적 제한 (잠시 후 재시도)
   → 대량 검사 시 발생 가능한 정상적인 현상
   → 10-15분 대기 후 다시 검사 실행

9. 오프라인에서 악성코드 검사 불가
   → 현재 버전은 인터넷 연결 필수 (온라인 DB 조회 방식)
   → 향후 업데이트에서 오프라인 검사 기능 추가 예정
   → 중요한 검사는 인터넷 연결 후 실행 권장

10. 네트워크 실시간 모니터링 실패
    → Npcap 설치 상태 확인 (패킷 캡처 필수 구성요소)
    → Windows 방화벽에서 LogCheck 허용 설정
    → 네트워크 어댑터 드라이버 최신 버전 업데이트
    → 가상 네트워크 어댑터가 아닌 실제 어댑터 선택

11. "사용 기록이 표시되지 않습니다"
    → Windows 감사 정책 활성화 여부 확인 (프로그램에서 자동 설정)
    → 이벤트 로그 보존 정책 확인 (기본 20MB 제한)
    → 감사 정책 활성화 이후 발생한 네트워크 활동만 기록됨
    → 시스템 로그 크기 늘리기: 이벤트 뷰어 → Windows 로그 → 보안 → 속성

12. 네트워크 모니터링 성능 저하
    → 장시간 모니터링 시 정상적인 현상 (메모리 사용량 증가)
    → 필요 시 캡처 중지 후 데이터 정리
    → 고성능 모드에서는 주기적 재시작 권장
    → 모니터링 대상 인터페이스 선택적 사용

13. "패킷 캡처 권한 없음" 오류
    → 반드시 관리자 권한으로 실행
    → Npcap 서비스 실행 상태 확인
    → 윈도우 방화벽 예외 설정 확인
    → 바이러스 백신에서 네트워크 모니터링 허용 설정

14. 보안 복구 실행 실패
    → 관리자 권한으로 실행 확인 (필수 조건)
    → PowerShell 실행 정책 확인: Set-ExecutionPolicy RemoteSigned
    → Windows 업데이트 최신 상태 유지
    → 바이러스 백신에서 PowerShell 스크립트 실행 허용

15. "Security Center 복구 실패" 오류
    → Windows 보안 센터 서비스 수동 시작: services.msc → Security Health Service
    → 시스템 파일 검사 실행: sfc /scannow
    → DISM 복구 실행: DISM /Online /Cleanup-Image /RestoreHealth
    → 필요 시 Windows 보안 센터 재설정

16. "복구 과정에서 응답 없음" 문제
    → PowerShell 명령 실행 중 정상적인 현상 (최대 2-3분 소요)
    → 바이러스 백신이 PowerShell 스크립트를 차단하는지 확인
    → 시스템 리소스 부족 시 다른 프로그램 종료 후 재시도
    → 필요 시 시스템 재시작 후 복구 재실행

17. PowerShell 출력 메시지 한글 깨짐
    → 시스템 지역 설정을 한국어로 변경
    → PowerShell 인코딩 설정: chcp 65001
    → 고급 모드에서 영문 메시지로 확인 가능
    → 향후 업데이트에서 UTF-8 인코딩 강제 적용 예정

18. "복구 완료"되었지만 여전히 문제 발생
    → 시스템 재부팅 후 보안 상태 재확인
    → Windows 업데이트 설치 후 재복구 시도
    → 일부 변경사항은 재부팅 후 적용됨
    → 복구 결과 보고서에서 실제 성공/실패 상태 확인
